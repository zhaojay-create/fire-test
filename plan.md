# Collision Layer 设计方案

## 项目：fireTest（Godot 4.6 / 2D）

---

## Layer 分配

| Layer | 名称          | 说明                           |
| ----: | ------------- | ------------------------------ |
|     1 | Player        | 玩家物理体                     |
|     2 | Monster       | 怪物物理体                     |
|     3 | PlayerAttack  | 玩家技能攻击（飞剑、环绕剑等） |
|     4 | MonsterAttack | 怪物攻击（弹幕、近战判定等）   |
|     5 | Environment   | 环境地形障碍（墙壁、岩石等）   |
|     6 | Pickup        | 掉落物（经验球、金币等）       |
|     7 | AreaEffect    | 领域效果（冰狱领域等）         |

---

## 各实体的 Layer / Mask 配置

### 1. 玩家 — `CharacterBody2D`

| 属性                | 值         | 说明                           |
| ------------------- | ---------- | ------------------------------ |
| **collision_layer** | 1 (Player) | 自身所在层                     |
| **collision_mask**  | 2, 5       | 与怪物产生物理碰撞；被地形阻挡 |

> 玩家不需要 mask 4（怪物攻击），因为怪物攻击是 Area2D，由攻击方主动检测玩家。

### 2. 怪物 — `CharacterBody2D`

| 属性                | 值          | 说明                                         |
| ------------------- | ----------- | -------------------------------------------- |
| **collision_layer** | 2 (Monster) | 自身所在层                                   |
| **collision_mask**  | 1, 2, 5     | 与玩家物理碰撞；怪物之间互相推挤；被地形阻挡 |

> 如果不需要怪物互相推挤，可去掉 mask 2。

### 3. 玩家技能攻击 — `Area2D`

| 属性                | 值               | 说明               |
| ------------------- | ---------------- | ------------------ |
| **collision_layer** | 3 (PlayerAttack) | 自身所在层         |
| **collision_mask**  | 2                | 检测怪物，触发伤害 |

> 对应场景：`sword_auto`（自动飞剑）、`sword`（环绕剑）等。
> 当前 sword 使用 `body_entered` 信号检测 `CharacterBody2D`（怪物），只需 mask 怪物层即可。

### 4. 怪物攻击 — `Area2D`

| 属性                | 值                | 说明               |
| ------------------- | ----------------- | ------------------ |
| **collision_layer** | 4 (MonsterAttack) | 自身所在层         |
| **collision_mask**  | 1                 | 检测玩家，触发伤害 |

### 5. 环境地形障碍 — `StaticBody2D` / `TileMap`

| 属性                | 值              | 说明                     |
| ------------------- | --------------- | ------------------------ |
| **collision_layer** | 5 (Environment) | 自身所在层               |
| **collision_mask**  | —               | 静态体通常不需要主动检测 |

> 地形只需被其他物体检测到（玩家 mask 5、怪物 mask 5），自身不需要 mask。

### 6. 掉落物 — `Area2D`

| 属性                | 值         | 说明                       |
| ------------------- | ---------- | -------------------------- |
| **collision_layer** | 6 (Pickup) | 自身所在层                 |
| **collision_mask**  | 1          | 检测玩家进入范围，触发拾取 |

> 掉落物主动检测玩家（layer 1），通过 `body_entered` 信号实现自动拾取或吸附效果。

### 7. 领域效果 — `Area2D`

| 属性                | 值             | 说明                   |
| ------------------- | -------------- | ---------------------- |
| **collision_layer** | 7 (AreaEffect) | 自身所在层             |
| **collision_mask**  | 1, 2           | 检测范围内的玩家和怪物 |

> - 玩家领域（冰狱等）：mask 只需 2（怪物）。
> - 怪物领域（毒雾等）：mask 只需 1（玩家）。
> - 若领域同时影响双方，则 mask 1 + 2。
> - 可根据需要进一步拆分为 layer 7（玩家领域）和 layer 8（怪物领域）。

## 备注

- **SwordAuto**（`sword_auto.tscn`）：当前为 `Area2D`，通过 `body_entered` 检测怪物。应设为 layer 3 / mask 2。
- **Sword**（`sword.tscn`）：同上，`Area2D` + `body_entered`，设为 layer 3 / mask 2。
- 若后续需要子弹穿透地形 / 被地形阻挡，可为 PlayerAttack 或 MonsterAttack 额外 mask 5。
- Layer 8–32 预留给未来扩展（如 NPC、友方召唤物、特殊触发区域等）。

# 游戏设计：修仙 Roguelike

## 类型定位

修仙题材 + Roguelike + 自动战斗（类吸血鬼幸存者）

---

## 一、核心循环

```
探索机缘 → 承受代价 → 短期暴涨 → 长期反噬 → 渡劫结算
```

核心原则：**机缘必带风险**，没有白拿的好处。

| 机缘类型 | 短期收益     | 长期代价                       |
| -------- | ------------ | ------------------------------ |
| 灵药     | +50% 修为    | 叠加"丹毒"（后期随机走火入魔） |
| 功法     | 快速变强     | 锁定发展路线（build 绑定）     |
| 法宝     | 强力主动技能 | 反噬神识（压缩神识上限）       |

---

## 二、境界系统

参考《凡人修仙传》境界体系：

```
炼气 → 筑基 → 结丹 → 元婴 → 化神
```

每个大境界 = Roguelike 的一个阶段，突破 = 阶段 Boss + Build 重构。

### 突破机制

突破不是简单升级，而是一次 **转职 + 遗产继承**：

- **清空**：部分技能/装备失效
- **保留**：道基、根骨、因果（永久词条）
- **重选**：基于当前积累，选择新的流派分支

### 流派分支示例（剑修路线）

```
炼气·剑修
  ├→ 筑基·纯剑    → 极致单体爆发
  ├→ 筑基·剑阵    → 多剑控场
  └→ 筑基·剑+雷   → 高风险高伤害
```

---

## 三、寿元系统

寿元 = 天然的回合限制，逼迫玩家做取舍。

### 基本规则

- 每局初始寿元：**120 年**
- 所有行动消耗时间（年）：

| 行动     | 时间消耗 | 收益       | 风险                |
| -------- | -------- | ---------- | ------------------- |
| 闭关修炼 | 10 年    | +修为      | 无                  |
| 探秘境   | 5~20 年  | 高收益机缘 | 重伤 → 额外消耗恢复 |
| 渡劫突破 | 即时     | 境界提升   | 失败 = 重伤/死亡    |

- 寿元归零 = 本局结束（坐化）

---

## 四、战斗系统：神识调度 + 自动战斗

战斗为自动战斗（类吸血鬼幸存者），玩家通过 **神识分配** 控制出战手段。

### 神识机制

- 神识 = 资源上限（初始 100）
- 每个技能/手段占用固定神识，**开启时占用，关闭时释放**（不是持续消耗）

| 手段         | 神识占用 | 说明           |
| ------------ | -------- | -------------- |
| 飞剑 ×1      | 10       | 每把剑独立占用 |
| 法宝（持续） | 20       | 开启后持续生效 |
| 阵法         | 30       | 领域控制效果   |
| 灵宠         | 30       | 自动追踪攻击   |

### 神识计算示例

```
全部开启：100 - 10 - 20 - 30 - 30 = 剩余 10
关闭飞剑：100 - 20 - 30 - 30     = 剩余 20
```

### 神识透支

- 神识可以为 **负值**（强行超载）
- 代价：**每秒消耗寿元**，透支越深消耗越快
- 设计意图：允许短时间爆发，但必须快速调整，否则寿元急剧流失

---

## 五、系统联动总结

```
神识透支 ──→ 消耗寿元 ──→ 缩短本局时间
   ↑                          ↓
法宝反噬 ──→ 压缩神识上限    寿元不足
   ↑                          ↓
吃灵药   ──→ 丹毒累积  ──→  走火入魔（强制渡劫/暴毙）
```

**核心博弈**：玩家在 "更强的当下" 和 "更长的未来" 之间不断抉择。
